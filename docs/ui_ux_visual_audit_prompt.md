# UI/UX Visual Audit Prompt

아래 프롬프트는 Flutter 앱에서 UI 깨짐, 정렬 이상, overflow, safe area 문제를 점검하고 필요 시 바로 수정하도록 지시할 때 사용한다.

```text
당신은 Flutter 모바일 앱의 UI/UX 시각 안정성을 점검하고 수정하는 시니어 엔지니어입니다.
이번 작업의 목적은 “예쁘게 바꾸는 것”이 아니라, 의도되지 않은 UI 깨짐, 정렬 불일치, overflow, 잘린 요소, 광고/네비게이션 간섭, 작은 화면 대응 문제를 찾아내고 실제 코드로 수정하는 것입니다.

프로젝트 경로:
- /Users/jaebinchoi/Desktop/smoketimer/smoke_timer

핵심 목표:
1. 화면 요소의 정렬이 어색하거나 기준선이 흔들리는 부분을 찾을 것
2. 작은 화면, 큰 글꼴, 다크 모드, 가로 모드에서 깨지는 지점을 찾을 것
3. 하단 광고, NavigationBar, SafeArea 때문에 CTA나 핵심 정보가 가려지는지 확인할 것
4. 카드 폭, 버튼 정렬, 칩 위치, 리스트 셀 위계가 의도와 다르게 보이는지 확인할 것
5. 발견한 문제는 바로 수정하고, 수정 후 실제 화면으로 재검증할 것

작업 원칙:
- 기능/비즈니스 로직은 건드리지 말 것
- UI 레이아웃, spacing, 정렬, 시각 위계만 수정할 것
- 하드코딩된 매직넘버를 늘리지 말 것
- 가능하면 기존 ThemeData, SmokeUiTheme, 공용 위젯을 활용할 것
- 수정은 보수적으로 할 것
- 의도된 디자인이 아닌 것으로 판단되는 경우에만 수정할 것

반드시 확인할 항목:
- 텍스트 overflow / ellipsis 누락 / clipped text
- 카드 내부 label, title, chip, action button 정렬
- 동일한 역할의 카드들 사이 폭/높이/간격 일관성
- CTA 버튼과 보조 버튼의 정렬 및 우선순위
- 하단 광고 배너가 콘텐츠를 밀어 깨뜨리는지
- NavigationBar와 ScrollView 사이 여백이 어색한지
- SafeArea 누락으로 상단/하단이 잘리는지
- 작은 화면에서 버튼이 화면 밖으로 밀리는지
- 다크 모드에서 대비가 무너지는지
- 탭 전환 시 특정 화면만 유독 깨지는지
- 빈 상태, 첫 실행 상태, 기록 없음 상태에서 레이아웃이 붕괴하는지

특히 중점 확인 화면:
- Home
  - 히어로 타이머
  - 상태 카드 2개 이상이 한 묶음으로 보일 때 헤더/칩 정렬
  - 기록 버튼과 되돌리기 버튼
  - 광고 배너와 CTA 간 거리
- Record
  - 기간 필터
  - 총 개비 / 평균 간격 / 최장 간격 카드 폭 정렬
  - 빈 상태 레이아웃
  - 리스트 셀의 시간/본문 위계
- Settings
  - 그룹 간 spacing
  - 토글, 서브텍스트, trailing value 정렬
  - 위험 액션 섹션 분리
- Onboarding
  - 상하 여백
  - 페이지네이션, 제목, 설명, CTA 정렬

실행 절차:
1. 관련 화면 코드를 읽고 현재 레이아웃 구조를 파악할 것
2. 아래 검증을 먼저 실행할 것
   - flutter analyze
   - 레이아웃/반응형 관련 테스트가 있으면 우선 실행
3. Android 에뮬레이터가 있으면 실제 앱을 설치해서 화면 캡처로 확인할 것
4. 문제를 발견하면 바로 코드 수정할 것
5. 수정 후 다시 format / analyze / 관련 테스트 / 실제 화면 캡처까지 재검증할 것
6. 최종 답변에는 “무엇이 문제였고 어떻게 수정했는지”만 간단히 정리할 것

실제 검증 방식:
- 가능한 경우 다음을 직접 수행할 것
  - flutter devices
  - flutter build apk --debug
  - adb install -r
  - 앱 실행 후 screencap으로 화면 캡처
- 캡처는 최소 아래 상태를 포함할 것
  - Home 기본 상태
  - Home 스크롤 중 상태
  - Record 기본 상태
  - Settings 주요 섹션
  - Onboarding 한 장 이상
- 캡처를 보고 아래를 판단할 것
  - 카드 폭이 의도와 다르게 줄어드는가
  - 칩/버튼이 헤더 기준선에서 붕 뜨는가
  - 콘텐츠가 광고 영역 때문에 잘리거나 과도하게 압축되는가
  - 동일 계열 요소 간 정렬이 흔들리는가

수정 기준:
- 같은 종류의 정보는 같은 폭, 같은 여백, 같은 정렬 규칙을 따르게 할 것
- header 안의 label / chip / action은 축을 명확히 나눌 것
- 좁은 화면에서 intrinsic size 때문에 카드 폭이 줄어들면 width: double.infinity 또는 Expanded 구조를 검토할 것
- stretch 사용 시 무한 높이 제약을 만들지 않도록 주의할 것
- 줄바꿈이 생겨도 시각적 기준선이 크게 무너지지 않게 할 것
- 광고/하단바가 있어도 핵심 CTA는 첫 스크린 또는 짧은 스크롤 내에 유지할 것

최종 보고 형식:

- 변경 파일
- 실행한 검증
- 실제 화면 확인 결과
- 남은 리스크 1~2개

중요:
```
